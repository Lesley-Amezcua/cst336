The Waterfall Model

The waterfall model is a linear sequential flow.
With this model progress is seen as flowing steadily downwards (like a waterfall) through the phases of software implementation. In this specific model phases in the development process begin only if the previous phase is complete. The waterfall approach does not define the process to go back to the precious phase to handle changes in requirement. This was the earliest approach that was used for software development.



Advantages Disadvantages
  • Helps to plan the project.
  • Each stage ensured early detection of errors.
  • Each phase has specific output.
  • Difficult to go back to any stage after it has finished.
  • It is costly and required more time, in addition to the detailed plan.
  • Flexibility and adjusting scope if difficult and expensive.
